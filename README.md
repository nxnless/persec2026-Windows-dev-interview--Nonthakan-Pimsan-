ข้อ 1 หลักการคือใช้ stack เก็บค่าที่เป็น open tag ไว้ กรณีที่เจอค่าที่เป็นคู่ของมันให้ pop ค่าออก โดย ความเร็วจะอยู่ที่ O(n) เพราะจะ Loop เก็บค่าแค่รอบเดียว โดบที่ n = จำนวน sub string ที่ input เข้าไป

ข้อ 2 เป็น natural Sort ข้อนี้ในมุม problem solving คิดไม่ออกคับ เลยทำ sort ธรรมดามา แต่มีข้อมูลที่ใช้ Comparer , NaturalSort.Extension ในเน็ตอยู่คับ

ข้อ 3 ข้อนี้ผมจะเริ่มจาก การloop เช็คก่อนว่า คำไหนบ้างที่ไม่มี คำที่ค้นหา แล้วจึง เอามาเข้าเช็คว่า คำที่ค้นหาอยู่ตำแหน่งไหนของคำ โดยจะไล่เช็คที่ละตำแหน่ง ของแต่ละคำ หากเจอจะเก็บไว้ แล้วลบข้อมูลเก่าทิ้ง ป้องกันการ search ซ้ำ


ข้อ 4 ค่าแต่ละหลักที่ได้จะ = num / หน่วยของเลขโรมัน เช่น 3000 / 1000 จะได้ MMM โดยหลังจาก เก็บค่าหน่วยที่มากที่สุดแล้วจะหารเก็บ เศษ (%) เพื่อคำนวณหน่วยต่อไป ความเร็วอยู่ที่ O(m) โดย m คือจำนวนค่าของเลข roman ที่บันทึกไว้

ข้อ 5 สามารถแก้โจทย์ได้โดยการเปลี่ยนค่า int -> string -> char[] , แล้ว sort desc แล้วค่อยเปลี่ยนกลับเป็น int โดยการเปลี่ยนค่าแต่ละครั้งจะใช้ความเร็วที่ O(n) และการ sort ใช้ O(nlogn) เป็น O(nlogn) + O(nlogn) + O(n) = O(nlogn) 

ข้อ 6 จะใช้ function ง่ายๆในการ บวก เลข โดยจะเริ่มหาตั้งแต่ จำนวนท้ายที่สุดก่อน จนถึงจำนวนเป้าหมายที่ต้องการ การบวกแต่ละครั้ง จะให้ O(1) เพราะรู้ว่าต้องบวกเลขอะไรบ้างอยู่แล้ว และ ความเร็วโดยรวมจะใช้ O(n) โดย n คือจำนวนเป้าหมายที่ต้องการ
